# 개발 로드맵

본 로드맵은 nutrition-platform을 단계적으로 구축하기 위한 순서를 정의합니다. 각 단계는 이전 단계의 산출물을 전제로 합니다. 현재는 1단계(초기 구조/문서화)만 진행되었습니다.

## 1단계 — 현재 구조 분석
- 저장소 초기 구조 및 문서(`docs/`) 정비 (현재 단계)
- 이관 대상 기존 시스템(AI 영양상담 매니저, 맛마을 탐험소, 상담신청/보호자동의) 코드 확보 및 인벤토리 작성
- `current-system-analysis.md` 기준으로 하드코딩된 값, Apps Script 함수, 데이터 흐름 파악

## 2단계 — Google 로그인과 최초 설치
- Google OAuth 연동 설계 및 구현 (영양교사 로그인)
- 최초 로그인 시 tenant 초기화 플로우 설계 (신규 학교 vs 기존 학교 판별)
- 인증/세션 관리 방식 결정 (Cloudflare Functions 기준)

## 3단계 — 학교별 시트/폴더 자동 생성
- 로그인한 Google 계정 기준으로 Google Sheets/Drive 자동 생성 로직 구현
- `database-schema.md` 기준 탭 구조 자동 생성
- `schoolPublicId` 발급 및 매핑 저장

## 4단계 — 영양상담 관리자 이전
- 기존 AI 영양상담 매니저 기능을 `/app` 모듈로 이관
- Sheets/Drive 연동을 신규 아키텍처(테넌트별 저장소) 기준으로 재구성

## 5단계 — 맛마을 탐험소 이전
- 기존 맛마을 탐험소 기능을 `/explore` 모듈로 이관
- 학생 대상 공개/비공개 접근 범위 재설계

## 6단계 — 상담신청 이전
- 기존 상담신청 기능을 `/intake` 모듈로 이관
- 비로그인 공개 폼 + token/`schoolPublicId` 기반 학교 식별 적용

## 7단계 — 보호자동의 이전
- 기존 보호자동의 기능을 `/consent` 모듈로 이관
- 상담신청과의 연결(`intakeId`, `caseId`) 및 동의 이력 관리 적용

## 8단계 — Gemini API 설정
- 관리자 설정(`/setup`)에서 학교별 Gemini API Key 등록/관리 기능 구현
- API Key 보관 방식을 보안 원칙에 맞게 설계 (서버 측 안전 저장, 브라우저 미노출)

## 9단계 — Cloudflare 자동 배포
- Cloudflare Pages/Functions CI/CD 파이프라인 구성
- 브랜치/환경별(개발·운영) 배포 전략 수립
- 배포 시 환경변수(민감정보) 관리 체계 확정

---

각 단계 착수 전, 해당 단계의 세부 작업 항목과 완료 기준(Definition of Done)을 별도로 정리하여 진행합니다. 현재 문서는 순서와 범위만 정의하며, 실제 구현은 포함하지 않습니다.
